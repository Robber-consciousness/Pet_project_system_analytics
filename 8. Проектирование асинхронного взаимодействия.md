
# 8.1 Асинхронное взаимодействие

 

В системе реализовано асинхронное взаимодействие между SCADA, Integration Service, Kafka и сервисами обработки инцидентов.
 После обнаружения аномалии SCADA отправляет событие в Kafka, откуда сообщения асинхронно обрабатываются сервисами уведомлений, логирования и аналитики.

 

### Сценарий:

 

При обнаружении аномалии данные от PLC и Sensors публикуются в Kafka-топик для последующей обработки и уведомления персонала.

 

### Технология:  Apache Kafka

  

# 8.2 Обоснование выбора технологии

 

Apache Kafka - это распределённая платформа потоковой обработки данных и брокер сообщений для высоконагруженных систем.

 

### Причины выбора Kafka:

 

-  Kafka обеспечивает обработку большого количества событий в реальном времени от PLC и IoT Sensors (до 10 000+ сообщений в минуту). 
-  Kafka позволяет автоматически распределять нагрузку между consumer-сервисами за счёт partition и consumer groups. 
-  Kafka обеспечивает отказоустойчивость и хранение событий, что позволяет использовать систему логирования и повторной обработки сообщений. 
-  Kafka хорошо подходит для промышленной платформы, где требуется асинхронное взаимодействие между SCADA, MES, аналитикой и сервисами уведомлений. 

 

# 8.3 AsyncAPI спецификация

 

Была написана спецификация AsyncAPI версии 2.6.0, использующая Apache Kafka для обмена сообщениями между компонентами цифровой платформы (Листинг 8.1).

  

```yaml
asyncapi: '2.6.0'

info:
  title: Industrial Monitoring Platform
  version: '1.0.0'

servers:
  production:
    url: kafka:9092
    protocol: kafka

channels:

  anomaly.detected:

    publish:
      summary: Событие обнаружения аномалии

      message:
        payload:
          type: object

          properties:
            sensorId:
              type: string

            timestamp:
              type: string

            severity:
              type: string

            description:
              type: string

  incident.created:

    subscribe:
      summary: Создание инцидента

      message:
        payload:
          type: object

          properties:
            incidentId:
              type: string

            status:
              type: string
```
